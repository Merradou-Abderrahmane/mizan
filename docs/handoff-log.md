**Decisions made during implementation:**
- The runner CLI contract is `php bin/runner <repoPath> [--composer=<path>] [--workdir=<path>]` → one JSON object on stdout, logs to stderr, exit codes 0/1/2. Symfony Process backs the subprocess checks.
- `Runner::run()` returns `[exit, json]` and the bin does the `fwrite(STDOUT)` — this keeps `Runner` unit-testable without output-buffering trickery (fwrite bypasses ob_start on Windows).
- Heavy fixtures are real `laravel/laravel` apps, gitignored, regenerated by `bin/setup-heavy-fixtures.ps1` + `bin/setup-mutated-fixtures.ps1`. Lightweight fixtures are committed.
- `laravel_readme_stub.md` is the committed byte-comparison anchor (the bytes of a freshly created Laravel app's default `README.md`).
- `MigrationsRunCheck` overrides Laravel's DB connection via process env vars (`DB_CONNECTION`, `DB_DATABASE`) — does NOT mutate the student's `.env`. It is non-`final` to expose a `protected function isSqliteAvailable()` subclass seam for the missing-extension skip test.
- `broken_bootstrap_repo` actually fails BOTH `composer_install` and `app_boots` (composer's `post-autoload-dump` scripts run `artisan`, which throws on bootstrap). This is realistic and intended; documented in `tests/fixtures/README.md`.
- On Windows, host `php` lacks the `zip` ext in `php.ini`, so the runner's composer/artisan invocations and the setup scripts invoke `php -d extension=php_zip.dll` or the Laragon `composer.phar` to work around dist downloads. Tests use the host `composer` binary; on already-installed fixtures `composer install` is a fast no-op and the missing zip ext is harmless inside the test run.
